# Душа проекта

`route_optimization` - это учебно-прикладной сервис для планирования городских маршрутов доставки. Его главная идея проста: диспетчер задаёт набор точек на карте, система строит маршрут через них, затем показывает более короткий порядок обхода и считает выигрыш по расстоянию и времени.

Проект не пытается быть полноценной промышленной TMS-системой. Он сфокусирован на понятном демо-сценарии: есть точки доставки, есть порядок посещения, есть метрики маршрута, есть визуальная карта.

## Для кого

Основной пользовательский образ - оператор или диспетчер небольшой службы доставки. Ему важно быстро увидеть маршрут, понять длину пути, сравнить исходный и оптимизированный варианты и принять решение без ручного расчёта.

Для команды проекта это ещё и демонстрационный продукт: он показывает умение собрать backend, frontend, БД, Docker, тесты и алгоритмическую часть в единый рабочий контур.

## Что проект уже умеет

- Генерировать случайные точки в радиусе вокруг заданного центра.
- Хранить текущий набор точек и маршрутов в SQLite.
- Строить базовый маршрут в порядке переданных точек.
- Оптимизировать порядок точек алгоритмом ближайшего соседа.
- Отображать точки и маршрут на Leaflet-карте.
- Показывать расстояние, время, источник маршрутизации и тип геометрии.
- Сбрасывать текущие точки и связанные маршруты.

## Осознанные ограничения

- Нет пользователей, аккаунтов и авторизации.
- Нет разделения данных между сессиями: приложение фактически рассчитано на один рабочий набор точек.
- Нет адресного ввода и геокодирования: пользователь работает с координатами.
- Оптимизация жадная, через Nearest Neighbor, поэтому не гарантирует глобальный оптимум.
- В env-шаблонах основным provider выбран `osrm`, чтобы показывать маршруты по дорогам.
- Если OSRM недоступен, приложение использует `haversine` fallback с прямыми линиями между координатами.

## Тон проекта

Проект лучше воспринимать как честный прототип маршрутизации: он уже демонстрирует полный путь от UI до сохранения и расчёта, но часть инженерной зрелости ещё впереди. Его ценность не в том, что он решает всю логистику, а в том, что он ясно показывает основу такой системы.
