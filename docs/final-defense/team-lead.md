# Team Lead задачи перед финальной защитой

Цель роли: собрать проект в финальную поставку: понятный README, публичный деплой, презентация, демо-сценарий и синхронизированная документация.

## Текущее состояние

Кодовая база близка к MVP, но финальная упаковка ещё не закрыта:

- README обновлён под локальный запуск, production flow, env, алгоритмы и ограничения;
- публичная ссылка на frontend/backend не зафиксирована;
- финальная презентация в `docs/` не найдена;
- `docs/project-state/` синхронизирована с текущим OSRM/fallback состоянием.

## Задачи

### TL-FINAL-01 `[TL][CRITICAL][DONE]` - полноценный README по ТЗ

Заменить текущий README на финальную версию.

Критерий готовности:

- краткое описание продукта;
- стек: FastAPI, SQLite, SQLAlchemy, Leaflet, Docker Compose, OSRM/haversine;
- архитектурная схема `browser -> nginx -> FastAPI -> SQLite -> OSRM`;
- инструкция запуска `docker compose up --build -d`;
- ссылки на frontend/backend/docs;
- таблица env-переменных;
- описание алгоритма Nearest Neighbor и статуса 2-opt;
- объяснение OSRM, haversine и fallback;
- ограничения: один пользователь, без auth, координаты вместо адресов, внешняя зависимость OSRM;
- публичная ссылка на деплой после TL-FINAL-02.

### TL-FINAL-02 `[TL][CRITICAL][TODO]` - публичный деплой

Довести deploy до состояния, которое можно показать комиссии.

Критерий готовности:

- публичный frontend URL открывается без VPN;
- backend `/health` доступен публично и возвращает `{"status":"ok"}` или актуальный health-response;
- frontend на публичном URL строит маршрут и не ходит в localhost;
- `CORS_ALLOW_ORIGINS` содержит публичный frontend origin;
- ссылки внесены в README и презентацию.

## Что обязательно сказать на защите

- MVP решает задачу одного курьера и одного маршрута.
- Реализованы генерация точек, базовый маршрут, оптимизация порядка, карта, метрики и порядок посещения.
- OSRM используется для маршрута по дорогам; haversine нужен как fallback и как приближённая модель.
- Ограничения известны: нет авторизации, нет адресного ввода, нет окон времени и вместимости, Nearest Neighbor не гарантирует глобальный оптимум.
