# Route Optimization — сводка по проекту

> Актуальная обзорная сводка. Основной входной документ для запуска и деплоя — `README.md`; задачи перед защитой ведутся в `docs/final-defense/`.

## Что это за продукт

Веб-приложение для планирования и оптимизации маршрутов доставки. Пользователь задаёт набор точек на карте, строит маршрут через все точки в произвольном порядке, затем одним кликом получает оптимизированный маршрут — такой, где суммарный путь короче. Приложение показывает, на сколько километров и минут удалось сократить маршрут.

Целевая аудитория: операторы логистики / диспетчеры небольших служб доставки, которым нужно быстро выстроить обход нескольких адресов.

## Реализованный функционал

### Генерация точек
- Пользователь задаёт **координаты центра** (широта/долгота), **радиус в км** и **количество точек**.
- Backend генерирует точки случайным образом равномерно внутри круга (формула с корнем из случайного расстояния — без скопления в центре).
- Точки сохраняются в БД, отображаются маркерами на карте.
- Новая генерация **полностью заменяет** предыдущие точки и маршруты.

### Базовый маршрут
- Строится по точкам **в том порядке, который выбрал пользователь** (порядок кликов или порядок генерации).
- Расчёт расстояния: формула гаверсинусов (прямое расстояние между точками).
- Скорость движения: фиксированные 40 км/ч → время в минутах.
- Маршрут рисуется на карте, показываются: расстояние (км), время (мин), провайдер, тип геометрии.

### Оптимизированный маршрут
- Алгоритм **ближайшего соседа (Nearest Neighbor Heuristic)**: из стартовой точки каждый следующий шаг — к ближайшей ещё не посещённой точке.
- После оптимизации показывается **сравнение с базовым маршрутом**: сколько км и минут сэкономлено, процент улучшения.
- Если оптимизированный маршрут оказался длиннее — об этом тоже сообщается.

### Порядок посещения
- После построения/оптимизации показывается список точек в порядке посещения с координатами.
- На карте каждая точка подписана номером в маршруте.

### Сброс
- Кнопка «Сбросить точки» удаляет все точки и маршруты из БД и с карты.

### Карта
- Leaflet + OpenStreetMap tiles.
- Маркеры точек, подписи с порядковым номером в маршруте, полилиния маршрута.
- При переключении между базовым и оптимизированным маршрутом — подсветка соответствующей карточки рамкой.

### Статус-индикатор
- Отображает текущее состояние: «Готово» / «Загрузка» / «Ошибка API».

## Провайдеры маршрутизации

Система поддерживает два провайдера, переключаемых через переменную окружения `ROUTING_PROVIDER`.

| Провайдер | Описание | По умолчанию |
|-----------|----------|--------------|
| `osrm` | Реальные дороги через публичный OSRM API | ✅ |
| `haversine` | Прямолинейные расстояния по формуле гаверсинусов, скорость 40 км/ч | fallback |

**Fallback-механизм**: класс `RoutingRouter` — если основной OSRM-провайдер недоступен, автоматически переключается на haversine. Фронтенд отображает предупреждение «routing API недоступен, использован приближённый расчёт».

## Что не реализовано / ограничения

- **Нет аутентификации** — приложение полностью открытое.
- **Один пользователь** — нет разделения данных между сеансами или пользователями.
- **Алгоритм оптимизации — жадный** (nearest neighbor). Не гарантирует глобальный оптимум, но работает быстро. 2-opt сейчас не реализован.
- **OSRM зависит от внешнего API** — если публичный OSRM недоступен, приложение использует haversine fallback.
- **Нет ввода адресов** — только координаты (lat/lon). Геокодирование не реализовано.
- **Нет экспорта** — маршрут нельзя сохранить в файл или отправить куда-либо.
- **Нет истории** — предыдущие маршруты доступны через API, но UI показывает только текущий сеанс.
