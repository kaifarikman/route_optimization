# Глоссарий

## Point

Точка доставки на карте. В backend представлена как `id`, `lat`, `lon`. Хранится в таблице `points`.

## Route

Маршрут через набор точек. Включает порядок ID точек, координаты, расстояние, время, геометрию линии и сведения о provider.

## Coordinates

Координаты исходных точек маршрута в формате `[lat, lon]`. Это не обязательно полная линия движения, а скорее список посещаемых точек.

## Geometry

Линия маршрута для отрисовки на карте. Для `haversine` это обычно точки маршрута, соединённые прямыми. Для полноценного OSRM-сценария geometry должна содержать много промежуточных координат дороги.

## Provider

Источник расчёта маршрута. В проекте есть `haversine` и `osrm`.

## Haversine

Формула расчёта кратчайшего расстояния между двумя координатами на сфере. В проекте используется как fallback provider и строит прямые линии между точками.

## OSRM

Open Source Routing Machine. Внешний routing API, который строит маршруты по реальным дорогам. В проекте OSRM provider реализует `build_route` и возвращает полную дорожную geometry.

## Fallback

Запасной сценарий, когда основной routing provider недоступен. Для `ROUTING_PROVIDER=osrm` factory создаёт `RoutingRouter`, который переключается на haversine и помечает маршрут `is_fallback=true`.

## Nearest Neighbor

Жадный алгоритм оптимизации порядка точек. Он начинает со стартовой точки и каждый раз выбирает ближайшую ещё не посещённую точку. Быстрый, понятный, но не гарантирует глобально оптимальный маршрут.

## 2-opt

Алгоритм улучшения маршрута через разворот участков пути. В текущем коде не реализован, но упоминается в задачах как следующий шаг после Nearest Neighbor.

## Unit of Work

Паттерн управления транзакцией. В проекте `SqlAlchemyUnitOfWork` создаёт session, repositories и отвечает за `commit` / `rollback`.

## Repository

Объект доступа к данным. `PointRepository` и `RouteRepository` скрывают SQLAlchemy-модели от сервисного слоя.

## Geometry type

Поле маршрута, которое показывает тип линии:

- `straight` - прямые линии;
- `full` - полная дорожная геометрия.

Для OSRM используется `full`, для haversine fallback используется `straight`.
